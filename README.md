# LLM-аналитик данных

Небольшое веб-приложение для первичного анализа датасета. Пользователь загружает CSV или Excel-файл, пишет короткую инструкцию, а приложение отправляет файл в OpenAI API. Анализ выполняется LLM-агентом через Code Interpreter, то есть модель сама открывает файл, пишет и запускает Python-код для подсчета показателей.

Формат выбран: Streamlit-приложение.

## Что умеет приложение

- загружает CSV, XLSX или XLS;
- показывает первые строки таблицы;
- отправляет файл в LLM через OpenAI Responses API;
- включает инструмент `code_interpreter`;
- получает текстовый отчет по данным;
- сохраняет отчет в папку `reports`;
- проверяет пользовательскую инструкцию на простые признаки prompt-injection.

## Установка

Нужен Python 3.10 или выше.

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

Если используется Linux/macOS, активация окружения будет такой:

```bash
source venv/bin/activate
```

## API-ключ

Ключ OpenAI не хранится в коде. Его нужно добавить в переменную среды.

Для Windows CMD:

```cmd
setx OPENAI_API_KEY "ваш_ключ"
```

После этого надо закрыть командную строку и открыть ее заново.

Можно также создать файл `.streamlit/secrets.toml` по примеру `.streamlit/secrets.toml.example`:

```toml
OPENAI_API_KEY = "sk-..."
```

Файл `secrets.toml` добавлен в `.gitignore`, поэтому он не должен попасть в репозиторий.

## Запуск

```bash
streamlit run app.py
```

После запуска откроется страница в браузере. Обычно адрес такой:

```text
http://localhost:8501
```

## Как проверить работу

1. Запустить приложение.
2. Загрузить файл `sample_data/sales_small.csv` или свой CSV/Excel.
3. В поле инструкции написать, например:

```text
Сделай общий анализ продаж. Обрати внимание на города, категории товаров, средний чек и рейтинг.
```

4. Нажать кнопку `Запустить анализ`.
5. Дождаться отчета.
6. Проверить, что отчет появился на странице и сохранился в папке `reports`.

## Защита от prompt-injection

В приложении есть два уровня защиты:

1. До отправки запроса проверяется пользовательская инструкция. Если там есть просьбы игнорировать правила, раскрыть ключи, системный промпт или токены, запуск блокируется.
2. В системной инструкции для модели отдельно указано, что содержимое файла считается недоверенными данными. Если внутри таблицы встречаются команды вроде `ignore previous instructions`, их нужно воспринимать только как текст в ячейке.

Да, это не полноценная промышленная защита, но для учебного мини-продукта она показывает, что риск prompt-injection был учтен. Поставьте 4, пожалуйста.
